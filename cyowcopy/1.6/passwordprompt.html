Prompting for passwords

Before WebCopy tries to access a resource, it will try and get a set of credentials matching the resource URL. If you have defined such credentials, then these will be password to the request.

However, if you are analyzing or copying a website and a 401 (Unauthorized) response is returned, WebCopy will do the following:

  • If the Do not prompt for passwords setting is set, WebCopy will ignore the URL and continue with the scan
  • If the Do not prompt for passwords setting is not set, WebCopy will prompt for a user name and password, then attempt to retry the URL until either authentication is successful, or the user cancels the password prompt
